This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Globalify is a Flask web app that connects entrepreneurs and investors. It pairs a PostgreSQL/SQLAlchemy database (source of truth) with Typesense (search index), uses server-rendered Jinja templates progressively enhanced with Vue 3 (loaded from CDN, not bundled), and styles everything with TailwindCSS compiled via PostCSS.

## Commands

Python is managed with `uv`; the venv must be active and a `.env` file present for most commands.

```bash
# Run the app (loads .env, activates venv, starts Flask on :5000)
source start.sh            # Mac/Linux
.\start.ps1                # Windows

# Tailwind watcher (run in a separate terminal during dev)
npm run css

# Format HTML (Prettier + tailwindcss plugin)
npm run html

# Lint / format Python
ruff check . --fix
ruff format .

# Recreate DB + seed demo data + rebuild Typesense indexes (run in shell, NOT flask shell)
flask setup

# Interactive DB access
flask shell                # then: db.drop_all() / db.create_all() / Investor.sync_search_index(True)

# Typesense schema setup via CLI
python -m src.project.utils.typesense_helpers.typesense_cli setup
```

Run Typesense locally with Docker (required for search):

```bash
docker run --name typesense -p 8108:8108 -v $(pwd)/typesense-data:/data \
  typesense/typesense:26.0 --data-dir /data --api-key=xyz --enable-cors
```

Tests: `pytest` is configured as a dev dependency but there is currently no test suite in the repo.

## Architecture

**App factory** — `src/project/__init__.py` defines `create_app()` and exports `application` (the WSGI entrypoint used by granian in the `Dockerfile`). It registers all blueprints, error handlers, OAuth providers (Google, LinkedIn, Apple), and Flask extensions. The `setup` CLI command also lives here.

**Extensions** — `src/project/extensions.py` holds the shared singletons (`db`, `login_manager`, `oauth`, `migrate`, `csrf`, `toolbar`). Import these rather than re-instantiating. SQLAlchemy uses the modern `DeclarativeBase` (`Base`) with typed `Mapped[...]` columns; many models use `MappedAsDataclass`.

**Blueprints** (`src/project/routes/`) map to URL prefixes registered in the factory: `auth` and `main` (no prefix), `claim`, `search`, `/payment`, `/settings`, `/profile`, `/admin`, `/onboarding`, `/investment`. Admin routes are further split under `routes/admin/`.

**Models** (`src/project/models/`) — re-exported from `models/__init__.py`. Core domain objects: `User`/`UserInfo`/`UserPayment`/`Company`, `Investor`/`InvestmentFirm` (with `InvestorBackup`, bookmarks, `NotableInvestment`), `Investment`/`FundingRound`, `ClaimRequest`/`ClaimVerification`, `SearchHistory`. `models/helpers.py` holds lookup tables (`Country`, `Industry`, `Round`).

**Search (Typesense)** — the key dual-store pattern: Postgres is the source of truth, Typesense is a derived search index. Models like `Investor` and `InvestmentFirm` define `sync_search_index(recreate=False)`, which (re)creates the collection schema — including vector `embedding` fields generated by Typesense's built-in `ts/all-MiniLM-L12-v2` model — and batch-uploads documents. After changing searchable model fields, the Typesense schema and the sync method must be updated together, then re-synced. `utils/typesense_helpers/typesense_search.py` provides the `client` and a fluent `SearchBuilder` (query/filter/sort/pinned/hidden hits) used by search routes.

**Frontend** — Jinja templates in `templates/` extend layouts in `templates/layouts/` (`layout_app`, `layout_auth`, `layout_clean`, `layout_error`, `layout_payment`). Reusable modals/widgets live in `templates/components/`. Vue 3 is loaded from CDN per-layout and component definitions live in `static/vue/*.js` — these progressively enhance server-rendered pages (e.g. fetch-based POSTs that read the CSRF token from a `#csrf_token` hidden input and send it as the `X-CSRFToken` header). CSS is compiled from `static/src/input.css` to `static/css/main.css`.

**Access control** — `utils/decorators.py` provides route guards: `admin_only`, `check_user_info_complete` (redirects to onboarding), `check_verification` (redirects to email verification).

**External integrations** — Google Cloud Storage (`utils/google_helpers/google_storage.py`) for file uploads, Google Pub/Sub (`google_pubsub.py`) for an async email service, SendGrid, Stripe (payments), Sentry (errors), PostHog (analytics). All credentials come from environment variables, conventionally prefixed with `_` (e.g. `_DATABASE_URL`, `_TYPESENSE_API_KEY`, `_GOOGLE_OAUTH2_CLIENT_ID`).

## Conventions

- **Env vars**: app/secret config uses a leading-underscore prefix (`_DATABASE_URL`, `_SENTRY_DSN`, `_TYPESENSE_*`, `_*_OAUTH2_*`, `_PUBSUB_*`). Defaults exist for local dev (sqlite, Typesense `xyz` key).
- **Python style**: ruff with line-length 120, double quotes, target py3.12; lint rules `E,F,W,B,Q,I,N,UP`. `migrations/` and stubs are excluded. A pre-commit hook runs ruff lint + format.
- **Testing mode**: set `FLASK_ENV=testing` to enable `TESTING`, disable CSRF, and use `test_db.sqlite`.
- **JS formatting**: Prettier (VSCode extension) for `static/vue/*.js`; `npm run html` for templates.
